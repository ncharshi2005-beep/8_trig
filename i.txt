Steps and Configuration
1. Prepare the GitHub Repository

Ensure your Calculator.java is in a GitHub repository (e.g., Github_CalculatorApp).

2. Setup ngrok (To enable Webhooks on Localhost)

Open your terminal and run: ngrok http 8080.

Copy the Forwarding URL provided (e.g., https://random-id.ngrok-free.app).

3. Configure GitHub Webhook

In your GitHub Repo, go to Settings > Webhooks > Add webhook.

Payload URL: Paste your ngrok URL followed by /github-webhook/ (Example: https://your-id.ngrok-free.app/github-webhook/).

Content type: application/json.

Click Add webhook.

4. Configure Jenkins Job Triggers

Go to your Jenkins job (from Experiment 7) and click Configure.

In the Build Triggers section:

Check GitHub hook trigger for GITScm polling.

Check Poll SCM.


Schedule: Enter * * * * * (this tells Jenkins to check every minute, though the Webhook will handle instant triggers).

Experiment 8: Jenkins Build TriggersAim:
To understand and configure Jenkins build automation by enabling build triggers such as SCM polling and GitHub webhooks in order to automatically trigger builds when changes occur in the source code repository.Theory:Build Trigger: Defines the condition under which a Jenkins job is executed.
SCM Polling: Jenkins periodically checks the repository for changes. If a change is detected, a build is triggered.GitHub Webhook: A real-time notification sent from GitHub to Jenkins. This is more efficient than polling as it triggers builds instantly upon a "push" event
.ngrok: Since your Jenkins is likely running on localhost:8080, GitHub cannot "see" it over the internet. 
ngrok creates a secure tunnel to expose your local server to the web, providing a URL that you can paste into GitHub's Webhook settings.
